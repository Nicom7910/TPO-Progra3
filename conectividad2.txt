CONSTRUCCIÓN DE RED MÍNIMA DE CONEXIONES

Modelaje del problema
La red social se modela como un grafo no dirigido y ponderado: G = (V, E) donde:
- V es el conjunto de usuarios.
- E es el conjunto de amistades.

Cada amistad se representa como una arista no dirigida (u, v).
Cada arista tiene un costo asociado, generado aleatoriamente entre 1 y 10.


En la simulación utilizada:
- Se consideran 10 usuarios.
- Se generan conexiones aleatorias entre ellos.
- Cada conexión tiene un costo aleatorio en el rango [1, 10].

Objetivo:
- Seleccionar el conjunto mínimo de conexiones necesarias para que todos los usuarios queden conectados entre sí, minimizando el costo total.
Este problema corresponde formalmente a la construcción de un Árbol de Recubrimiento Mínimo.

Estrategia utilizada:
- El algoritmo construye el árbol agregando conexiones en orden creciente de costo, siempre que no generen ciclos.
- Se divide en tres etapas.


ETAPA 1 — Ordenamiento de conexiones:
- Se ordenan todas las aristas del grafo en orden ascendente según su costo.
Esto garantiza que siempre se intente agregar primero la conexión más barata disponible.

ETAPA 2 — Construcción incremental usando Union-Find:
- Se inicializa una estructura Union-Find (Disjoint Set Union).
- Cada usuario comienza en su propia componente.
- Se recorren las aristas ordenadas por costo.

Para cada arista (u, v):
- Si u y v pertenecen a componentes distintas:
      Se agrega la arista al MST.
      Se realiza la operación union(u, v).
- Si ya pertenecen a la misma componente:
      Se rechaza la arista (formaría un ciclo).


El proceso continúa hasta que el árbol tenga |V| − 1 aristas.

ETAPA 3 — Finalización:
- Cuando el MST contiene |V| − 1 aristas:
    - Todos los usuarios quedan conectados.
    - No existen ciclos.
    - Se obtiene el costo total mínimo.
    - El algoritmo finaliza.

Pseudocódigo

Algoritmo KruskalMST
Entrada:
V = conjunto de usuarios
E = conjunto de conexiones con costo

Salida:
MST = conjunto de conexiones mínimas

Inicio

Ordenar E por costo ascendente
Crear estructura UnionFind(V)
MST ← ∅

Para cada arista (u, v) en E hacer

    Si Find(u) ≠ Find(v) entonces
        Agregar (u, v) a MST
        Union(u, v)
    FinSi

    Si |MST| = |V| − 1 entonces
        Terminar
    FinSi

FinPara
Fin


Justificación del algoritmo

El problema corresponde al caso clásico de Minimum Spanning Tree (MST).
El algoritmo de Kruskal es óptimo para este tipo de problemas porque:
- Siempre selecciona la arista más barata disponible.
- Evita ciclos mediante la estructura Union-Find.
- Se apoya en el Teorema del Corte, que garantiza que elegir la arista mínima entre componentes distintas no compromete la optimalidad.
- Cada decisión greedy es segura: elegir la arista más barata que conecta dos componentes nunca impide alcanzar la solución óptima global.
Por lo tanto, el algoritmo garantiza un costo mínimo total sin necesidad de explorar todas las combinaciones posibles.

Análisis de complejidad

Sea:
V = cantidad de usuarios
E = cantidad de conexiones

El algoritmo realiza:

- Ordenamiento de aristas => O(E log E)

- Operaciones Union-Find => O(E α(V))
donde α(V) es la función inversa de Ackermann, prácticamente constante.

Complejidad total => O(E log E)


En la simulación utilizada:
V = 10
E ≈ 18
El algoritmo ejecuta en tiempo prácticamente inmediato.
